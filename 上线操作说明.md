# 小红书填表网站上线操作说明

这份说明按最省事的 Render 方式写。做完以后，你和老婆在公司或家里都可以打开同一个网址使用。

## 你需要准备

1. 一个 GitHub 账号。
2. 一个 Render 账号：https://render.com
3. OpenAI 或兼容接口的 API Key。如果暂时没有，也可以先不填，系统会用本地规则解析二询。

## 第一步：把代码放到 GitHub

如果电脑还没有 Git，先安装：

```text
https://git-scm.com/download/win
```

安装后重开 Cursor 或 PowerShell，然后在项目目录运行：

```powershell
cd D:\1\Documents\xiaohongshu-ratecard-automation
git init
git add .
git commit -m "Deploy multiuser web app"
```

然后去 GitHub 新建一个仓库，建议选 Private。按 GitHub 页面提示执行类似下面的命令：

```powershell
git remote add origin https://github.com/你的账号/你的仓库名.git
git branch -M main
git push -u origin main
```

## 第二步：在 Render 创建网站

1. 打开 Render Dashboard。
2. 点击 New。
3. 选择 Blueprint。
4. 连接 GitHub，选择这个项目仓库。
5. Render 会自动读取 `render.yaml`。
6. 确认创建。

Render 会自动创建：

- 一个 Web Service。

当前仓库默认是「免绑卡先跑起来」配置，不会自动创建 PostgreSQL 和持久磁盘。如果后面要长期稳定保存数据，再升级到正式版数据库和持久磁盘。

## 第三步：补环境变量

部署创建后，进入 Render 的 Web Service，打开 Environment。

必须建议填写：

```text
OPENAI_API_KEY=你的模型 API Key
```

可选但建议填写：

```text
REGISTER_INVITE_CODE=你自己设一个邀请码，例如 xhs2026
```

设置邀请码后，只有知道邀请码的人才能注册账号。你可以把这个邀请码发给老婆。

`SECRET_KEY`、`DATA_DIR`、`DATABASE_PATH`、`STORAGE_DIR` 会由 `render.yaml` 自动配置，不需要你手动填。

## 第四步：打开网站注册

部署成功后，Render 会给你一个网址，类似：

```text
https://xiaohongshu-ratecard-automation.onrender.com
```

打开后：

1. 你先注册一个账号。
2. 老婆再注册一个账号。
3. 如果设置了邀请码，注册时填同一个邀请码。

每个账号的数据互相隔离。你看不到老婆的任务，她也看不到你的任务。

## 第五步：验收

用你的账号：

1. 上传刊例库。
2. 上传品牌 Excel 创建任务。
3. 搜达人补齐一行。
4. 粘贴二询解析。
5. 导出 Excel。

再用老婆账号登录：

1. 确认看不到你的任务。
2. 上传她自己的刊例和品牌表。
3. 确认可以导出。

## 常见问题

### 网站第一次打开很慢

Render 免费/低配服务冷启动可能需要几十秒，等一下刷新即可。

### 上传后重启会不会丢

当前免绑卡方案不保证永久保存。服务重启、Render 重新部署或免费服务回收时，任务数据和 Excel 文件可能丢失。先用它跑通流程；如果你们确定长期使用，再升级到 PostgreSQL 和持久磁盘。

### Render 让我填银行卡怎么办

先不要填，点关闭或返回。确认 GitHub 上的 `render.yaml` 已经是免绑卡版本后，再重新创建 Blueprint。免绑卡版本不会创建 Render PostgreSQL 和持久磁盘。

### 暂时没有 OpenAI Key 能不能用

可以。没有 `OPENAI_API_KEY` 时，二询解析会退回本地规则，但复杂句子的识别效果会差一些。

### 国内访问慢怎么办

先用 Render 跑通最省事。后面如果经常慢，再迁到阿里云或腾讯云轻量服务器。
