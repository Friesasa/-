# 小红书刊例自动生成工具

这个工具用于把蒲公英导出的博主数据 Excel，自动合并到现有小红书刊例 Excel 中。

当前版本只处理小红书刊例工作表 `5月刊例`，不会更新 `视频号` 工作表。

## Web 版自动填品牌表

现在也提供一个网站版 MVP，可以上传刊例生成达人库、上传品牌表格、按昵称/ID 补齐刊例基础信息、粘贴二询文字自动填表、标注缺失项并导出品牌 Excel。

本地启动：

```powershell
py -m pip install -r requirements.txt
py -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

然后打开：

```text
http://127.0.0.1:8000
```

更多说明见 `WEB_APP.md`。

## 已实现

- 读取蒲公英导出 Excel，自动识别第 3 行表头。
- 读取旧刊例 Excel 的 `5月刊例` 工作表。
- 根据 `博主ID`、`小红书号/账号ID`、主页链接匹配达人。
- 自动生成新的 `5月刊例` 主表。
- 旧刊例有、但本次蒲公英导出没有的达人，会移入 `下架留存` 工作表，不进入主刊例。
- 保留旧刊例人工维护字段：`合作笔记案例参考`、`返点`、`备注`、`机构`、`负责人`。
- `机构` 字段优先使用蒲公英导出的 `所属机构`。
- 主刊例只保留 `所属机构` 为空或为 `漫洵文化` 的达人；其他机构达人会进入 `其他机构留存`。
- 蒲公英导入中重复出现的达人，会进入 `重复达人`，主刊例默认保留第一条，并在 `备注` 中标记。
- 生成 `更新日志` 工作表，记录本次处理数量。

## 使用方式

### 方式一：双击运行

以后不需要再复制命令。按下面步骤操作：

1. 打开文件夹：

```text
D:\1\Documents\xiaohongshu-ratecard-automation\input
```

2. 把两个 Excel 放进去：

- 蒲公英导出的 Excel
- 旧刊例 Excel

3. 回到这个文件夹：

```text
D:\1\Documents\xiaohongshu-ratecard-automation
```

4. 双击：

```text
一键生成刊例.bat
```

5. 生成后的文件会自动放到：

```text
D:\1\Documents\xiaohongshu-ratecard-automation\output
```

注意：`input` 文件夹里建议每次只放这两个 Excel，避免脚本识别混乱。生成前请先关闭已经打开的输出 Excel，避免文件被占用。

### 方式二：命令运行

在 PowerShell 中运行：

```powershell
py "D:\1\Documents\xiaohongshu-ratecard-automation\generate_ratecard.py" `
  --pgy "蒲公英导出的Excel路径.xlsx" `
  --ratecard "旧刊例Excel路径.xlsx" `
  --output "输出的新刊例路径.xlsx"
```

本次样例已生成：

```text
D:\1\Documents\xiaohongshu-ratecard-automation\output\小红书刊例_自动生成_20260529_公式版.xlsx
```

## 每周更新建议

每周从蒲公英导出当前需要展示的达人列表，然后运行脚本。输出文件中的 `5月刊例` 就是本期展示表，`下架留存` 是本期不展示但保留历史的达人。

如果蒲公英导出的 `所属机构` 不是空、也不是 `漫洵文化`，该达人不会进入 `5月刊例`，会进入 `其他机构留存`。如果蒲公英导出里有重复达人，脚本会把重复记录写入 `重复达人`，主刊例默认保留第一条。

## 每月更新建议

每月用全量蒲公英导出文件运行脚本，价格、粉丝、CPM/CPE、粉丝画像等字段会整体刷新。人工字段仍从旧刊例保留。

## 后续接入飞书

接入飞书 API 后，脚本可以改成直接同步到飞书多维表格。需要你先准备：

- 飞书开放平台企业自建应用的 `App ID`
- 飞书开放平台企业自建应用的 `App Secret`
- 飞书多维表格的 `app_token`
- 多维表格中的表 ID：达人主库、蒲公英导入明细、刊例配置、小红书刊例结果

建议的飞书多维表格结构：

- `蒲公英导入明细`：保存每次导入的原始字段。
- `达人主库`：每个达人一条，记录当前状态、最近更新时间。
- `刊例配置`：维护负责人、返点、机构、案例链接、备注等人工字段。
- `小红书刊例结果`：展示最终刊例结果。
